# 90 术语表

术语表只收录跨章节反复出现、且容易被混用的词。定义优先服务于快速判断，不追求辞典式铺陈。

## 当前术语

- `NPU`：面向神经网络负载的专用加速器，目标通常是以更高能效执行矩阵乘加、卷积、注意力等算子。
- `PE`：Processing Element，阵列里的最小计算单元，通常包含乘加、寄存器和局部控制。
- `MAC`：Multiply-Accumulate，乘加，是神经网络推理中最常见的基础操作。
- `Dataflow`：规定权重、激活、部分和在阵列与存储层级中的流动方式，本质上决定复用方向和带宽压力。
- `DMA`：Direct Memory Access，不经主核逐项搬运而由专用搬运单元进行数据传输。
- `Runtime`：承接编译结果，在系统运行时完成装载、调度、同步、资源管理和异常处理的软件层。
- `Tile`：为了片上复用和局部执行而切出来的一小块工作集，大小同时受 SRAM、阵列粒度、DMA 粒度约束。
- `Weight Stationary`：一种数据流，目标是尽量让权重停留在低成本层级，减少权重重复搬运。
- `Output Stationary`：一种数据流，目标是让部分和尽量在本地完成累积，减少中间回写与重读。
- `PTQ`：Post-Training Quantization，训练后量化；重部署效率，代价是对敏感层更脆弱。
- `QAT`：Quantization-Aware Training，量化感知训练；把量化误差提前带入训练过程以稳住精度。
- `Zero-Skipping`：在稀疏计算中跳过零值参与的搬运或计算；是否真正赚到收益取决于编码、解码和负载均衡。
- `IR`：Intermediate Representation，中间表示；编译器在不同抽象层表达图语义、循环结构和目标工件的载体。
- `BYOC`：Bring Your Own Codegen；把候选子图标注、切分后交给外部 backend codegen，并把返回的 `runtime.Module` 接回主执行图。
- `Target Translation`：把低层 IR 分别翻译成 host 侧和 device 侧目标模块的阶段，是编译链进入具体硬件承诺的位置。
- `Lowering`：把高层图或循环表示逐层压低到更接近目标硬件执行形式的过程。
- `Codegen`：根据 lowering 后的表示生成目标指令、命令流、描述符或 runtime 工件的过程。
- `Model Cache`：缓存已经编译好的模型工件以减少首次启动时延；是否可复用取决于 plugin、driver、版本与设备边界。
- `Static Shape`：张量 shape 在编译阶段就固定的执行路径；很多 NPU 高效路径只在 static shape 下成立。
- `Fallback`：某段图或某个算子未走目标 NPU 高效路径，而回退到 CPU、GPU 或通用实现。

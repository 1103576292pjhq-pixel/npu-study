# 01 数字、向量、矩阵、乘加

## 1. 先看一个生活化例子

假设一杯饮料由三种原料组成：

```text
糖 2 份，奶 3 份，茶 4 份
```

现在有一个评分规则：

```text
糖每份 1 分，奶每份 2 分，茶每份 1 分
```

总分就是：

```text
2*1 + 3*2 + 4*1 = 2 + 6 + 4 = 12
```

这就是一次最小的“向量点积”：两排数字对应相乘，再把乘积加起来。

## 2. 直觉：神经网络里到处都是点积

神经网络判断一个东西，常常不是看一个数字，而是同时看很多特征。每个特征有一个权重，表示它重要不重要、正向还是反向。

如果特征是：

```text
[2, 3, 4]
```

权重是：

```text
[1, 2, 1]
```

那么点积结果是 12。换一组权重，就像换一种判断规则。NPU 大量做的事，就是把这类点积复制到很大的规模。

## 3. 正式定义

向量是一排有顺序的数字。两个长度相同的向量做点积，就是对应位置相乘后求和：

```text
[a0, a1, a2] dot [b0, b1, b2]
= a0*b0 + a1*b1 + a2*b2
```

矩阵是一张二维数字表。矩阵乘法可以看成“很多次行向量和列向量的点积”。

如果：

```text
A = [1 2]
    [3 4]

B = [5 6]
    [7 8]
```

那么输出 `C` 的每个位置都是一组点积。

## 4. 完整手算：2x2 矩阵乘法

先算 `C[0,0]`：A 的第 0 行和 B 的第 0 列点积。

```text
C[0,0] = 1*5 + 2*7 = 5 + 14 = 19
```

再算 `C[0,1]`：

```text
C[0,1] = 1*6 + 2*8 = 6 + 16 = 22
```

再算 `C[1,0]`：

```text
C[1,0] = 3*5 + 4*7 = 15 + 28 = 43
```

最后算 `C[1,1]`：

```text
C[1,1] = 3*6 + 4*8 = 18 + 32 = 50
```

所以：

```text
C = [19 22]
    [43 50]
```

每个输出都用了“乘、加、再乘、再加”。这就是 MAC 的来源。

## 5. 连接到 NPU

NPU 常把大量 MAC 单元排成阵列。矩阵乘法天然适合阵列化：

- 不同输出位置可以同时算。
- 同一行 A 会被多个输出复用。
- 同一列 B 也会被多个输出复用。
- 每个输出内部有一个累加过程，需要保存部分和。

一个小阵列可以这样理解：

```text
        B 列0     B 列1
          |        |
A 行0 --> PE ---- PE --> C 行0
A 行1 --> PE ---- PE --> C 行1
```

每个 PE 负责一小部分乘加，多个 PE 同时工作，就能比通用 CPU 一条条算更快。

## 6. 常见误区

- 误区一：矩阵乘只是数学课内容。对 NPU 来说，它是硬件阵列最核心的工作负载之一。
- 误区二：只要 MAC 多就一定快。如果 A、B 数据供不上，MAC 会空等。
- 误区三：点积只关心乘法。真正决定输出的是乘法和累加共同完成的归约。
- 误区四：矩阵越大只是循环次数变多。矩阵变大后，数据复用和片上存储也会变成关键问题。

## 7. 图解：一个输出值的来源

```text
A 的一行:  [1  2  3]
             |  |  |
             *  *  *
             |  |  |
B 的一列:  [4  5  6]

乘积:       4 10 18
求和:       4 + 10 + 18 = 32
```

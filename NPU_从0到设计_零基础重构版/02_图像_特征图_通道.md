# 02 图像、特征图、通道

## 1. 先看一张小图

黑白图像可以先理解成一张数字表。数字越大，像素越亮：

```text
0 0 0 0
0 9 9 0
0 9 9 0
0 0 0 0
```

这张 4x4 图中间有一个亮方块。计算机看不到“方块”这个词，它只看到 16 个数字。

## 2. 直觉：通道就是同一个位置上的多种描述

彩色图像不是一张表，而是三张表叠在一起：红色 R、绿色 G、蓝色 B。

```text
位置 (h,w)
  |
  +-- R 数字
  +-- G 数字
  +-- B 数字
```

这三层就是 3 个通道。神经网络中间的特征图也类似，只不过通道不一定是颜色，可能是“边缘强不强”“横线强不强”“某种纹理强不强”等抽象特征。

## 3. 正式定义

图像张量常写成 `H x W x C`：

- `H`：高度，有多少行。
- `W`：宽度，有多少列。
- `C`：通道数，每个位置有多少种数字描述。

特征图是神经网络某一层输出的张量。它仍然可以按高度、宽度、通道理解，只是通道含义从颜色逐渐变成学习出来的特征。

## 4. 完整手算：一个位置的多通道加权

假设某个像素位置有 RGB 三个数：

```text
R = 2, G = 5, B = 1
```

有一个简单规则想强调绿色、弱化蓝色：

```text
R 权重 1，G 权重 2，B 权重 -1
```

输出就是：

```text
2*1 + 5*2 + 1*(-1)
= 2 + 10 - 1
= 11
```

这说明同一个空间位置也可以做点积：把多个通道的值乘以对应权重，再加起来。

## 5. 图解：H、W、C

```text
一个 3x3x2 特征图

通道 0             通道 1
1 2 3              0 1 0
4 5 6              1 0 1
7 8 9              0 1 0

同一个 (行,列) 位置，有两个通道值。
```

如果取中心位置 `(1,1)`：

```text
通道 0 的值 = 5
通道 1 的值 = 0
```

这个位置的“描述”不是一个数，而是 `[5, 0]`。

## 6. 连接到 NPU

NPU 处理图像和特征图时，最关心的不只是总共有多少数字，还关心这些数字的形状：

- 高度和宽度决定滑动窗口会经过多少位置。
- 通道数决定每个输出值要累加多少层。
- 数据排布决定一次搬运能不能连续读到需要的数字。
- 特征图太大时，必须切成 tile 放进片上 SRAM。

同样是 `3x3x64` 的一块数据，如果通道方向在内存里连续，适合一种读取方式；如果宽度方向连续，又适合另一种读取方式。后面讨论数据流时，这些形状都会变成硬件选择。

## 7. 常见误区

- 误区一：通道只存在于彩色图片。神经网络中间层可能有 16、32、64 甚至更多通道。
- 误区二：特征图一定能被人直接看懂。越往后，通道含义越抽象。
- 误区三：只看总元素数就够了。`4x4x8` 和 `8x8x2` 总元素数一样，但滑动窗口和复用方式不同。
- 误区四：layout 只是软件格式。layout 会影响 NPU 读数据是否连续、是否容易并行。

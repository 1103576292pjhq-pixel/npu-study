# 04 卷积如何变成循环

## 1. 先把上一章的手算重复很多次

上一章算 3x3 输入和 2x2 卷积核，得到 2x2 输出。手算时我们做了四件事：

```text
算 O[0,0]
算 O[0,1]
算 O[1,0]
算 O[1,1]
```

每件事内部又做 4 次乘法和 3 次加法。图像更大、通道更多、卷积核更多时，人不可能一格格手算，程序就要用循环描述这件事。

## 2. 直觉：循环只是把“滑动和累加”写清楚

卷积的循环可以分成两类：

- 外层循环：决定现在算哪个输出位置、哪个输出通道。
- 内层循环：决定这个输出值内部要累加哪些输入和权重。

一个输出值不是一步得到的，而是从 0 开始，反复执行：

```text
部分和 = 部分和 + 输入值 * 权重值
```

直到窗口内所有位置和所有输入通道都累加完。

## 3. 正式定义：最朴素的卷积循环

对输入 `I[H][W][IC]`、权重 `K[OC][KH][KW][IC]`、输出 `O[OH][OW][OC]`，最直观的循环是：

```text
for oc in 0 .. OC-1:
  for oh in 0 .. OH-1:
    for ow in 0 .. OW-1:
      sum = 0
      for ic in 0 .. IC-1:
        for kh in 0 .. KH-1:
          for kw in 0 .. KW-1:
            sum += I[oh+kh][ow+kw][ic] * K[oc][kh][kw][ic]
      O[oh][ow][oc] = sum
```

这段伪代码就是卷积从数学公式变成计算任务的关键。

## 4. 完整手算：把一个输出翻译成循环

仍然使用输入：

```text
1 0 2
0 1 3
2 1 0
```

卷积核：

```text
1 0
0 1
```

计算 `O[0,1]` 时，`oh=0, ow=1`。循环访问的窗口位置是：

```text
I[0+0][1+0] = I[0][1] = 0
I[0+0][1+1] = I[0][2] = 2
I[0+1][1+0] = I[1][1] = 1
I[0+1][1+1] = I[1][2] = 3
```

按 `kh, kw` 循环展开：

```text
sum = 0
kh=0, kw=0: sum += 0 * 1 => 0
kh=0, kw=1: sum += 2 * 0 => 0
kh=1, kw=0: sum += 1 * 0 => 0
kh=1, kw=1: sum += 3 * 1 => 3
```

最终：

```text
O[0,1] = 3
```

这和上一章手算结果一致。

## 5. 为什么 NPU 不直接照软件循环慢慢跑

软件循环表达的是依赖关系，硬件要进一步决定怎么并行和复用。

例如这些事情可以同时或流水执行：

- 不同 `ow` 位置的输出可以分给不同 PE。
- 不同 `oc` 的卷积核可以分给不同 PE 组。
- 同一个输入窗口可以被多个输出通道共享。
- 同一个权重可以在多个空间位置复用。
- 部分和可以留在 PE 或局部 buffer 中，最后再写回。

因此 NPU 会把循环映射成硬件执行计划，而不是一条指令一条指令解释。

## 6. 图解：循环维度到硬件问题

```text
oc 循环  -> 用哪组权重，能否并行多个输出通道
oh/ow   -> 窗口滑到哪里，能否并行多个空间位置
ic      -> 输入通道归约，部分和放在哪里
kh/kw   -> 卷积核内部扫描，输入窗口怎样读取

所有循环共同决定：
数据搬运 + 阵列映射 + 部分和累加
```

## 7. 连接到 NPU

NPU 设计里的很多术语都可以从这段循环推出：

- `PE 阵列`：用来同时执行很多次 MAC。
- `硬件循环`：用专用控制器推进规则循环，减少通用指令开销。
- `tile`：当完整输入、权重或输出放不进片上存储时，把循环范围切成小块。
- `dataflow`：决定输入、权重、部分和谁固定、谁流动、谁复用。
- `DMA`：在计算当前 tile 时搬运下一个 tile，尽量不让阵列等待。

所以卷积变成循环之后，NPU 的任务就是让这些循环在硬件上跑得更并行、更少搬运、更少空等。

## 8. 常见误区

- 误区一：循环只是软件写法，硬件不关心。硬件正是根据循环依赖来安排并行、复用和存储。
- 误区二：把所有循环都并行展开最好。展开太多会耗费面积、带宽和片上存储。
- 误区三：tile 只是把大问题切小。tile 的核心目的是让数据能在片上被复用。
- 误区四：部分和可以随便写回。频繁写回和重读部分和会浪费带宽和能量。
